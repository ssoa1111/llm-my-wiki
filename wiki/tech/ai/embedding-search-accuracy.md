# 임베딩 검색 정확성 향상 전략

> 단순 벡터 유사도 검색의 한계를 넘어 AI 에이전트가 능동적으로 검색 품질을 개선하는 전략

## 핵심 내용

단순 벡터 검색은 쿼리 표현 방식에 따라 결과가 크게 달라진다. 이를 개선하기 위한 Multi-Step 전략:

### 핵심 전략 7가지

**1. Multi-Step Search (4단계)**
1. 쿼리 최적화: 긴 문장 → 핵심 키워드 + 동의어 고려
2. 검색 결과 해석: Confidence 레벨 기반 판단
3. 결과 불충분 시: 키워드 변형·주제 목록·대안 활용
4. 응답 생성: 출처 명시 + 신뢰도 표시

**2. Confidence 레벨 시스템**

| Confidence | 유사도 | 의미 | 대응 |
|------------|--------|------|------|
| high | ≥ 0.7 | 매우 관련성 높음 | 바로 결과 활용 |
| medium | ≥ 0.5 | 관련 있음 | 결과 + 대안 확인 |
| low | ≥ 0.3 | 약한 관련성 | 대안 확인 or 재검색 |
| none | < 0.3 | 관련 없음 | 키워드 변경 재검색 |

**3. 상위 N개 결과 + 대안 제공**
- 임계값 0.3으로 상위 5개 검색
- 1위: 주 결과 / 2-5위: 대안 주제
- AI가 1위만으로 부족할 때 대안에서 더 적합한 주제 선택 가능

**4. 낮은 임계값 설정 (0.3)**
- 기존 0.5보다 낮춰 더 많은 후보 수집
- Confidence 시스템으로 품질 관리
- False negative(관련 결과 누락) 방지

**5. 쿼리 최적화 가이드**
```
✅ 좋은 검색: "URL 구조", "메타태그"
❌ 나쁜 검색: "URL을 어떻게 구성해야 할까요?"

동의어 맵핑:
- URL = 주소, 경로, 링크
- 메타태그 = title, description, meta
```

**6. 검색 실패 시 복구 전략**
- 전략 A: 키워드 변형 (일반화, 동의어, 언어 변환)
- 전략 B: 전체 주제 목록 조회 후 선택
- 전략 C: alternativeTopics에서 재선택

**7. 도구 호출 횟수 제한 (최대 5회)**
- 무한 루프 방지 + 토큰 비용 절감
- 있는 정보로 최선의 답변 유도

### 핵심 인사이트

**"임베딩 검색 + AI 추론"의 시너지**: 단순 벡터 유사도 검색이 아니라 AI 에이전트가 능동적으로 검색 품질을 개선하는 구조. 첫 검색 실패 → 키워드 변형 재시도 → Confidence 낮음 → 대안 탐색 → 전체 목록 확인.
