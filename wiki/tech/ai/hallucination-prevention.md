# 할루시네이션 방지 및 개선

> LLM이 없는 정보를 지어내는 할루시네이션을 줄이는 프롬프트 설계, 생성 단계 분리, 출력 검증 전략.

## 핵심 내용

### 프롬프트 설계 원칙

**근거 의무화**
```
"모든 주장은 반드시 입력 데이터 또는 제공된 자료에서 근거를 제시하라.
출처가 없으면 '근거 없음'으로 표시하라."
```

**추측 제한**
```
"정보가 부족한 경우 내용을 만들어내지 말고, '정보 부족'이라고 답하라."
```

---

### 생성 단계 분리 (핵심 전략)

한 번에 다 시키면 환각 확률이 올라간다. 단계를 나누면 오류 탐지가 쉬워진다.

**데이터 분석 보고서 예시**:
```
1단계 (사실 추출):
"데이터셋에서 주요 지표의 월별 합계를 JSON으로 정리하라. 해석 금지."

2단계 (해석):
"이 JSON 데이터를 기반으로 Insight만 정리하라.
반드시 수치를 포함하고, 데이터 범위 안에서만 해석하라."

3단계 (보고서):
"이 Insight를 보고서 형식으로 재구성하라."
```

**지식 기반 Q&A 예시** (RAG 적용):
```
1단계: 벡터DB에서 관련 문서 검색 → JSON 출력
2단계: 문서에서 질문에 답하는 부분만 발췌 요약
3단계: 요약된 내용으로 답변, 각 문단에 출처 인용 필수
```

**마케팅 카피 예시**:
```
1단계: 제품 특징을 10가지 관점에서 설명 (bullet point)
2단계: 특정 톤앤매너로 변환
3단계: 상위 3개 선택해 슬로건 완성
```

---

### 출력 검증 전략

**사실 검증 레이어**: 결과물을 2차 모델로 "이 내용이 입력 근거와 일치하는지" 검증.

**일관성 체크**: 수치, 날짜, 이름을 정규식으로 추출해 원본 데이터와 매칭.

**교차 질문**: 같은 질문을 살짝 다르게 여러 번 물어보고 답변 불일치 시 경고.

---

### 모델 사용 전략

- **과잉 창의성 제한**: "사실 기반" 모드로 유도, 스토리텔링은 별도 단계로 분리.
- **작게 쪼개기**: "한 문단에서 인사이트 3개" 대신 "인사이트 1개만"을 반복 실행.
- **검증된 모델 선택**: 최신 Claude/GPT/Gemini 상위 모델일수록 할루시네이션 적음.
- **Self-RAG 적용**: LangGraph로 답변 생성 후 자동 검증·재생성 루프 구축.

---

### Self-RAG와의 관계

[Self-RAG](../../concepts/advanced-rag.md)는 할루시네이션 방지의 자동화된 구현이다. 답변 생성 후 ①문서 지원 여부 ②환각 여부 ③관련성을 병렬 검증하고, 실패 시 최대 3회 재생성. 환각률을 20-30%에서 5-10% 이하로 낮출 수 있다.
