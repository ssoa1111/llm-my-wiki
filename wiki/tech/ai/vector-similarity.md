# 벡터 유사도 측정

> 두 임베딩 벡터 간의 유사도를 계산하는 방법. RAG 검색 품질의 핵심.

## 핵심 내용

[벡터 데이터베이스](./vector-database.md)는 쿼리 벡터와 저장된 벡터들 사이의 유사도를 계산해 가장 관련 있는 문서를 반환한다.

### 주요 3가지 방식

**1. 코사인 유사도 (Cosine Similarity) ⭐ 표준**
- 벡터 간의 **각도**를 측정 (-1 ~ 1, 1이 가장 유사)
- 벡터 크기(길이)를 무시하고 **방향(의미)만** 비교
- 문서 길이에 무관하게 의미 비교 → 텍스트 임베딩에 최적
- **사용 비율: 90%** — 기본값으로 사용하면 됨

**2. 유클리드 거리 (Euclidean Distance)**
- 벡터 간 직선 거리 측정 (0 ~ ∞, 0이 가장 유사)
- 벡터 **크기까지 고려** — 절대적 차이가 중요할 때
- 이미지 임베딩에 적합
- **사용 비율: 8%**

**3. 내적 (Dot Product)**
- 벡터 곱셈 (-∞ ~ ∞, 클수록 유사)
- 정규화된 벡터에서는 코사인 유사도와 동일
- 속도가 빠름 → 추천 시스템에 활용
- **사용 비율: 2%**

### 직관적 비교

```
vec1 = [0.5, 0.5]  (짧은 벡터)
vec2 = [1.0, 1.0]  (긴 벡터, 같은 방향)

코사인: 1.0  → "방향이 같으니 같은 의미" ✅ (RAG에 적합)
유클리드: 0.7 → "길이 차이도 감지"
```

### 실무 결론

```
RAG/텍스트 검색  → 코사인 유사도 (특별한 이유 없으면 이걸로)
이미지 유사도    → 유클리드 거리
추천 시스템      → 내적 (속도 우선)
```

OpenAI, Cohere 등 주요 임베딩 모델 모두 코사인 유사도를 권장한다.
